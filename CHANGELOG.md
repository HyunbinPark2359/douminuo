## [Unreleased]

### 그 외 (트래픽·성능 최적화)

내부 동작 정리 — 사용자 화면에서 보이는 변경은 없습니다. 핵심은 **스마트누오 사이트로 보내는 불필요한 호출을 줄이는 것**입니다.

- **샘플 변환 시 누오 호출 정비.**
  - 출력에 URL이 들어가지 않는 옵션(`URL` 끔 또는 `Showdown` 형식)이면 **공유 등록 호출을 보내지 않습니다.** 예: Showdown 형식으로 파티 복사 시 옛날엔 누오에 8건이 갔지만 이제 0건입니다.
  - 파티 복사 직후 동일 데이터를 다시 받아오던 GET 단계를 제거했습니다. 파티 복사당 누오 호출 1건 절감.
  - 비어 있는 슬롯이 누오에 등록 요청으로 새는 케이스를 잡았습니다. 부분 파티(예: 3마리 + 3빈)에서 빈 슬롯 3건 절감.
- **데미지 계산기 자동 입력 첫 사용 시 발생하던 PokéAPI 일괄 다운로드 제거.** 이미 빌드 타임에 들고 있는 한글 표기 매핑(`moveKoMap.json`)을 그대로 쓰도록 정리해, 이전엔 처음 사용자가 ~10~30초 가량 기다려야 했던 단계가 **즉시 끝납니다**.
- **같은 공유 URL로 60초 이내 재시도 시 누오 GET을 한 번만**. 이전엔 매 클릭마다 GET이 갔습니다.
- **결정력 계산 시 PokéAPI 의존 제거**. 종족 타입을 외부에서 받지 않고, 빌드 타임 번들 (`extension/pokemonTypeMap.json`)에서 lookup 합니다. 신규 폼 등장 시에만 PokéAPI 1회 + 캐시.
- **확장 자체 번들 JSON 의 SW 라이프타임 캐시.** 같은 `chrome-extension://...json` 파일을 한 동작 안에서 여러 번 또는 클릭마다 다시 파싱하던 동작을 한 번으로 줄였습니다. 동작은 변하지 않고 SW 부담만 줄어듭니다.
- **팀빌더 인라인 결정력·내구력 갱신 사이클의 SW 메시지 절감.** 페이지 측 가드로 어차피 noop이던 bridge 재주입 메시지를 첫 1회로 한정.
- **간단 스피드 계산기 알고리즘 정리.** 0..500 선형 탐색을 닫힌형 + ±2 검증으로 교체. 200,000건 이상 입력 조합에서 옛 결과와 동일함을 확인했습니다 (사용자에게는 표시 변화 없음).
- **킬가르도 폼 변환 시 결정력 표기 정확도 보강.** 슬롯 개체값을 우선 사용, 없을 때만 31 가정.
- **팀빌더 인라인 결정력·내구력의 갱신 사이클이 가벼워졌습니다.** 사이트의 Vue 데이터 트리를 매 사이클마다 전수 traversal 하던 패턴을 짧은 캐시로 대체했고, 임시 객체 생성도 줄였습니다. 화면에 보이는 동작은 변하지 않습니다.
- **계산기 화면 감지 비용 절감.** 페이지 본문 텍스트를 매번 다시 읽으며 발생하던 layout 강제 flush 를 250ms 짧은 캐시로 대체했습니다.
- **간단 스피드 계산기 idle CPU 0.** 환경설정에서 기능을 끄면 이제 진정한 idle 상태가 됩니다 (이전엔 200ms 주기 폴이 빈 채로 계속 돌고 있었음). 켜진 상태에서도 입력 wrap을 캐시해 매 tick의 비용을 줄였습니다.
- **슬롯 평탄화 함수 alias 정렬.** Service Worker 와 콘텐츠 스크립트의 슬롯 처리 함수가 약간 다른 alias 를 갖고 있어 새 슬롯 모양이 들어올 때 한쪽에서만 누락될 위험이 있었습니다. 두 구현 모두 동일 alias 셋을 갖도록 정리.
- **환경설정 변경 감지 코드 통합.** 콘텐츠 스크립트 4곳에서 같은 모양의 `chrome.storage.onChanged` 보일러플레이트를 반복하던 것을 공용 헬퍼 한 줄로 정리.
- **에러 메시지 매핑 dict 화.** 에러 코드 → 한국어 메시지 매핑을 `if-체인` 대신 객체 리터럴로. 새 코드 추가 시 한 줄로 끝.
- **레귤레이션 스피드 표 stale 가드.** 개발자가 `regulationMaSpeedTable.json` 편집 후 `embedSpeedData.js` 재실행을 잊으면 콘솔 경고로 알려 줍니다. 사용자에게는 영향 없음.
- **팀빌더 콘텐츠 스크립트 파일 분리.** 우하단 FAB 와 좌측 슬롯 카드 인라인 어노테이션이 한 파일(2046줄)에 들어 있던 것을 두 시스템으로 분리. 동작 변화 없음.

배경: 스마트누오는 비영리·사비로 운영되는 사이트입니다. 본 확장이 어머니 사이트의 트래픽을 늘리는 일을 최소화하기 위한 정비입니다.
