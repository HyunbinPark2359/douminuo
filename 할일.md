## Showdown 붙여넣기 — PokeAPI 한글 없음 폴백

**원인:** PokeAPI `names`에 `language: ko`가 없으면 `moveKoMap.json` / `itemKoMap.json` 생성 시 해당 항목이 빠짐 → 한글 그대로 출력.

**처리:** `extension/moveKoFallback.json`, `itemKoFallback.json`에 **누오에 보이는 한글 키 → Showdown id / item slug** 를 넣고, 팝업 로드·`generate-move-ko-map.js`에서 **기존 맵에 없을 때만** 덧씀.

**공식 명칭 검증 (웹 서치 허용 후):** 아래는 **누오 표기 추정 → 기술/도구 id** 연결. 한글판·발매처 표기와 다르면 JSON 키를 고치고, PokeAPI에 `ko`가 생기면 해당 폴백 줄은 **삭제**.

- [ ] `찬물끼얹기` → `chillingwater`
- [ ] `라이트닝드라이브` → `electrodrift` (다른 기술이면 id 교정)
- [ ] `펀치글러브` → `punching-glove`

**전수 갱신:** `node scripts/audit-pokeapi-ko-gaps.js` → `scripts/pokeapi-ko-gaps-report.json` (갭 요약).

**PokeAPI 본맵 일괄 재생성:** `node scripts/refresh-ko-maps-from-pokeapi.js`  
→ `moveTags.json` → `moveKoMap.json` → `itemKoMap.json` → `abilityKoMap.json` → `moveSlugToEn.json` (순서 고정, 네트워크·수분 소요).

개별만: `generate-move-tags.js` / `generate-move-ko-map.js` / `generate-item-ko-map.js` / `generate-ability-ko-map.js` / `generate-move-slug-to-en.js`.

누오에서 한글로 튀면 폴백에 **누오 표기 → id/slug** 추가: `moveKoFallback.json`, `itemKoFallback.json`, `abilityKoFallback.json`.

## 보류 / 후순위 (덩어리 큼)

- **내구력(옵션):** README「내구력 — 구현 범위(확정)」까지 **완료**. 추가 특성·도구는 그 절·`modifiers.json` 기준으로 수동 확장.
- [ ] **결정력을 “전부” 표시** — 위력뿐 아니라 특성·지닌물건에 따른 위력 보정까지 거의 다 모델링해야 해서 부담 큼(내구는 위 확정 범위와 별개).  
  - 조사: `GET/POST /api/party/share`만으로는 부족한지 확인됨. 스마트누오에서 데미지 UI 쓸 때 **별도 API가 있는지** DevTools Network로 확인.  
  - 대안: 별도 API가 없으면 `@smogon/calc` 등 데미지 엔진 번들 + 공유 JSON → calc 입력 매핑(설계·매핑 표는 미작성). 로컬 JSON만으로 예외 규칙 전부 유지는 어렵다.

- [ ] **실수치 한 줄에서 안 쓰는 능력치를 `x`로** — 예: 물리기만 쓰는 따라큐의 특공 실수치. 종만으로는 판정 불가(루카리오·누리레느 등). **기술배치 기준** 판정 + 기술별 물리/특수 데이터 필요 → 후순위.

## 결정력 모델 — 더 넣을 후보 (체크리스트)

**참고 (이미 넣은 쪽):** `modifiers.json`·`moveTags.json`·`moveKoMap.json` 기준으로 A/B/C 일부·부모의애착 등 반영. 스킬링크·연속타 합산·플래시파이어 미포함. D·E·G·H는 의도적 제외.

**아직 안 넣은 것·모델 밖:** 플래시파이어, 불칸 전용 타입 상징, 테라전개 추가 보정, 분석·선제·라이벌리·최고대장, D(상성·급소), E(배터리·파워스팟·아군 강철의마음), G(슬로스타트·패배주의), H(받는 쪽·명중만), 연속기마다 위력 다른 기술 정밀 재현.

- [ ] 분석·선제·라이벌리·최고대장 (턴·교체·성별·파티 상태)
- [ ] 플래시파이어 (활성 여부 데이터 필요)
- [ ] 배터리·파워스팟 등 더블 전용
- [ ] 상성·급소 계열
- [ ] 슬로스타트·패배주의 등 디버프
- [ ] 연속기별 위력이 다른 기술(트리플악셀 등) 정밀 모델
