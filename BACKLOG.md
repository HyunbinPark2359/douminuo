# 숙제 / 백로그

나중에 손대거나 대안을 찾을 항목만 짧게 적어 둔다.

## 1. 결정력·내구력 표시

기술별 결정력, 포켓몬의 물리·특수 내구력 등을 넣으려면 위력뿐 아니라 **특성·지닌물건**에 따른 위력·내구 보정까지 경우의 수를 거의 다 알아야 해서 리소스·노력 대비 부담이 큼. 보류.

**데이터 소스 (정리):** 지금 확장이 쓰는 `GET/POST /api/party/share` 응답만으로는 특성·도구 배율이나 미리 계산된 결정력/내구력이 내려오지 않는 것으로 보인다. (1) 스마트누오 사이트에서 데미지 UI 사용 시 **별도 API가 있는지** DevTools Network로 확인 — [docs/smartnuo-network-audit.md](docs/smartnuo-network-audit.md). (2) 없으면 **Smogon 계열 `@smogon/calc`** 등 데미지 엔진을 번들해, 공유 JSON을 calc 입력으로 매핑하는 쪽이 현실적 — [docs/damage-calc-integration.md](docs/damage-calc-integration.md). “배율 전부 로컬 JSON”만으로는 예외 규칙 유지가 어렵다.

## 2. 실능치 줄에서 쓰지 않는 능력치를 `x`로

예: 물리 공격기만 쓰는 따라큐의 특수공격 실능치를 `x`로 두는 식.

포켓몬 종만으로는 판정이 어렵다. 루카리오처럼 물·특 양면 빌드, 누리레느처럼 특수 쪽인데 퀵턴·아쿠아제트 등 **물리 판정 기술**을 쓰는 경우 등. 결국 **기술배치 기준**으로 판정해야 하고, 그러려면 기술마다 물리/특수 구분 데이터가 필요해 리소스·effort가 크다. **후순위**.

## 3. 단순 결정력(위력×STAB×실수치…) — 특성 상정 후보 종합

아래는 **“내 포켓몬이 때릴 때” 숫자에 들어가는 특성** 위주다. (상대 방어·급소·상성 표를 안 쓰는 현재 모델 기준.)  
**이미 `modifiers.json` + `moveTags.json`으로 반영된 것**은 제외하고, **3차에서 고를 수 있게** 남은 것만 나열한다.

### A. 타입·위력 직접 배율 (공유 데이터에 **기술 타입·종 타입**만 있으면 상당 부분 가능)

| 특성 (영문 slug 참고) | 요약 | 비고 / 필요 데이터 |
|----------------------|------|-------------------|
| 적응력 (adaptability) | 동일 타입 보정을 1.5→2.0에 가깝게 | STAB 단계에 `stabMul` 또는 동일 규칙 추가 |
| 맹화 / 심록 / 급류 / 벌레의알림 (blaze, overgrow, torrent, swarm) | HP 1/3 미만일 때 해당 타입 위력 1.5배 | **현재 HP%** 또는 스마트누오에 비율 필드 |
| 용의턱 (dragons-maw) | 드래곤 기술 위력 1.5배 | 이도 타입만 |
| 강철의마음 (steely-spirit) | 자신의 강철 기술 1.5배 (더블에서 아군에게도) | 단일 슬롯만 보면 “자신”만 반영 가능 |
| 트랜지스터 (transistor) | 전기 기술 위력 보정 (세대·패치별 배율 상이) | 공식·세대 고정 필요 |
| 강철일꾼 (steelworker) | 강철 기술 1.5배 | 이도 타입만 |
| 어둠의오라 (dark-aura) | 암커 기술 위력 약 1.33배 (필드; 더블에서 상대에게도) | 날씨급 **전역 플래그** 없으면 “항상 켜짐”으로 과대평가 |
| 페어리오라 (fairy-aura) | 페어리 기술 동일 | 위와 동일 |
| 모래의힘 (sand-force) | 모래바람 중 땅·바위·강철 위력 1.3배 | **날씨** |
| 플래시파이어 (flash-fire) | 활성화 시 불꽃 위력 상승 | 공유에 **활성 여부** 없으면 추정 불가 |
| 물거품 (water-bubble) | 물 타입으로 주는 데미지 상승(세대 문구 상이) | 보통 “물 기술 배율”로 근사 가능 |
| 불칸의상징 등 타입 고정 위력 특성 | 종·작품별 | 케이스마다 확인 |

### B. 노말→타입 변환 + 위력 보정 (-ate 계열)

| 특성 | 요약 | 비고 |
|------|------|------|
| 스카이스킨 (aerilate) | 노말→비행 + 위력 보정 | 실제 타입·위력 단계 둘 다 수정 |
| 폭신스킨 (pixilate) | 노말→페어리 | |
| 프리즈스킨 (refrigerate) | 노말→얼음 | |
| 갤바나이즈 (galvanize) | 노말→전기 | |
| 노말스킨 (normalize) | 전 기술 노말타입 | STAB·타입 플레이트와 상호작용 |

### C. 순서·교체·성별·필드 (공유 JSON에 **없으면** 거의 불가능하거나 큰 가정 필요)

| 특성 | 요약 | 필요 데이터 |
|------|------|------------|
| 분석 (analytic) | 후공이면 위력 1.3배 | 선후공·속도·트릭룸 등 |
| 선제 (stakeout) | 교체 직후 첫 턴 대상에게 2배 | 턴·교체 이력 없으면 불가 |
| 라이벌리 (rivalry) | 동성 1.25배 / 이성 0.75배 | **성별** (자신·상대) |
| 최고대장 (supreme-overlord) | 기절한 동료 수에 비례해 보정 | 파티 기절 수·남은 마리 수 |
| 솔파워 (solar-power) | 쾌청에 **특수공격** 상승(위력이 아니라 스탯) | 날씨 + 스탯 단계 반영 여부 |
| 테라전개류 / 기동력 / 오리카르펄스 등 | 날씨·지형·필드 | 필드 상태 |

### D. 상성·급소 계열 (전체 데미지식 쪽 — **단순 결정력 한 줄**과 결이 다름)

| 특성 | 요약 |
|------|------|
| 색안경 (tinted-lens) | “별로 효과가 없음”일 때 보정 |
| 신경력 (neuroforce) | 효과가 본 기술에 대해 추가 보정 |
| 스나이퍼 (sniper) | 급소 시 배율 변화 |
| 메르시레스 (merciless) | 중독 시 급소 등 |
| 일격 (one-hit-kill) 계열과의 조합 | — |

→ 이들은 **타입 상성표·급소 확률** 없이는 의미 있게 넣기 어렵다.

### E. 아군 전용 / 더블 우선 (단일 샘플 UI와 안 맞을 수 있음)

| 특성 | 요약 |
|------|------|
| 배터리 (battery) | 옆 아군의 특수기 위력 |
| 파워스팟 (power-spot) | 옆 아군의 물리기 위력 |
| (위 Steely Spirit) | 아군 강철기까지 |

### F. 다단히트·형태 특수 (현재 “한 방 위력×실수치”와 모델 불일치)

| 특성 | 요약 |
|------|------|
| 스킬링크 (skill-link) | 다히트를 최대 횟수로 고정 |
| 부모의사랑 (parental-bond) 등 | 2회 타격·배율 분할 |
| 연속기·기술 자체가 multihit | `moveTags`·별도 위력 합산 규칙 필요 |

### G. 공격 스탯을 깎는 특성 (숫자는 나오지만 “버프”가 아님)

| 특성 | 요약 |
|------|------|
| 슬로스타트 (slow-start) | 5턴 공격·속도 절반 등 |
| 패배주의 (defeatist) | HP 절반 이하에서 공·특 절반 |
| 겸손 (humbling) 등 | 작품별 |

→ **현재 HP** 또는 **턴**이 있으면 `atkMulPhysical` / `spaMulSpecial`로 넣을 수는 있음.

### H. 기타 (백로그에만 적어 두기)

- **받는 쪽** 특성(멀티스케일, 필터, 얼음가시, 두꺼운지방 등): “내 결정력”이 아니라 상대 기준.
- **명중·우선도만** 바꾸는 특성: 결정력 숫자와 별개.
- **기교·상태로만** 데미지가 바뀌는 경우: 별도 시스템.

### 3차 범위 정하기용 체크리스트 (exclude 하기 좋게)

- [ ] A만 (타입·STAB·날씨 없이 가능한 것만)
- [ ] A + 맹화계 (HP 조건만 추가)
- [ ] A + 모래의힘 / 오라계 (날씨·전역 가정 포함)
- [ ] B (-ate / 노말스킨) 포함 여부
- [ ] C (분석·선제·라이벌리·최고대장) 데이터 없으면 제외
- [ ] D (상성·급소) 이번 3차에서 완전 제외
- [ ] E (더블 전용) 제외
- [ ] F (다히트) 제외 또는 후순위
- [ ] G (디버프 특성) 포함 여부

이 섹션은 **참고용 종합 목록**이다. 3차에서 실제로 넣을 항목은 위 체크로 줄이면 된다.
