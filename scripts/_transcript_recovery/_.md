# 할 일 (숙제·과제)

아직 하지 않았거나, 나중에 조사·구현할 작업만 짧게 모아 둔다. (제품 백로그가 아니라 **우리 쪽에서 손댈 일** 목록.)

---

## 보류 / 후순위 (덩어리 큼)

- [ ] **결정력·내구력을 “전부” 표시** — 위력뿐 아니라 특성·지닌물건에 따른 위력·내구 보정까지 거의 다 모델링해야 해서 부담 큼.  
  - 조사: `GET/POST /api/party/share`만으로는 부족한지 확인됨. 스마트누오에서 데미지 UI 쓸 때 **별도 API가 있는지** DevTools Network로 확인.  
  - 대안: 별도 API가 없으면 `@smogon/calc` 등 데미지 엔진 번들 + 공유 JSON → calc 입력 매핑(설계·매핑 표는 미작성). 로컬 JSON만으로 예외 규칙 전부 유지는 어렵다.

- [ ] **실능치 한 줄에서 안 쓰는 능력치를 `x`로** — 예: 물리기만 쓰는 따라큐의 특공 실능치. 종만으로는 판정 불가(루카리오·누리레느 등). **기술배치 기준** 판정 + 기술별 물리/특수 데이터 필요 → 후순위.

---

## 결정력 모델 — 더 넣을 후보 (체크리스트)

**참고 (이미 넣은 쪽):** `modifiers.json`·`moveTags.json`·`moveKoMap.json` 기준으로 A/B/C 일부·부모의애착 등 반영. 스킬링크·연속타 합산·플래시파이어 미포함. D·E·G·H는 의도적 제외.

**아직 안 넣은 것·모델 밖:** 플래시파이어, 불칸 전용 타입 상징, 테라전개 추가 보정, 분석·선제·라이벌리·최고대장, D(상성·급소), E(배터리·파워스팟·아군 강철의마음), G(슬로스타트·패배주의), H(받는 쪽·명중만), 연속기마다 위력 다른 기술 정밀 재현.

- [ ] 분석·선제·라이벌리·최고대장 (턴·교체·성별·파티 상태)
- [ ] 플래시파이어 (활성 여부 데이터 필요)
- [ ] 배터리·파워스팟 등 더블 전용
- [ ] 상성·급소 계열
- [ ] 슬로스타트·패배주의 등 디버프
- [ ] 연속기별 위력이 다른 기술(트리플악셀 등) 정밀 모델
